## 3. CapacityReport 已完成的关键改动

### 3.1 数据目录统一映射

- 取消旧 `UDData` / `RJData` 概念，统一为 `DataMappings`。
- `DataMappings.directories` 每行：`{ path, table, ready_rule }`。
- `ready_rule=daily`：按目标周每日 7 天检查。
- `ready_rule=auto`：按 ZIP 最新文件自动识别日/周粒度。
- 表级字段映射：`DataMappings.table_field_mappings[table]`。

### 3.2 CellData 配置

- `CellData` 配置块包含：
  - FTP/SFTP 连接信息：`remote_data`
  - MySQL 连接信息：`mysql`
  - 扫描路径：`scan_paths`
  - 正则：`year_dir_regex` / `month_dir_regex` / `day_dir_regex` / `file_name_regex` / `file_time_regex`
  - 映射 JSON：`mapping`
- CellData 数据源卡片不再显示“远程目录”；连接根目录固定为 `/`，实际数据位置由扫描路径控制。
- 默认扫描路径：`/网优日常优化数据文档/日常性能报表/{maxyear}年/300表`
- 占位符：
  - `{maxyear}`：扫描同级年份目录取最大年份
  - `{maxmonth}`：扫描同级月份目录取最大月份
  - `{maxday}`：扫描同级日期目录取最大日期
  - `{yyyy}` / `{yyyymm}` / `{yyyymmdd}`：当前日期占位符

### 3.3 CellData 预处理

- 新增 `app/services/cell_data.py`。
- 做的事：
  - 解析扫描路径模板
  - 按扫描目录下一级子目录（如 `700M`、`2.6G`）选最新 `Result_300_*.zip`
  - 解压 ZIP
  - 只识别 `LTE_ITBBU_CellInfo`、`LTE_SDR_CellInfo`、`NR_CellInfo`
  - 按 `CellData.mapping` 映射字段
  - 生成 `CGI = PLMN-eNodeBID-CellID`
  - 清空并批量写入 `celldata.cellinfo`
- CellData 处理不会删除远程文件，任何情况下都不能删除，因为该 FTP/SFTP 数据源是共享数据。
- CellData 单独处理和容量处理共用现有全局任务锁。

### 3.4 CellData 前端入口

- 数据处理页有两个卡片：
  - 容量数据
  - CellData
- 两个卡片宽屏并排，窄屏单列。
- 两个卡片左上角有类型标识，右上角有说明按钮。
- 容量数据说明：启用 CellData 数据源时，处理容量数据前会先更新 CellData。
- CellData 卡片：
  - 点击卡片选择文件夹
  - 拖拽只接受文件夹
  - 不支持单个 ZIP 直接拖入
  - 保留“远程下载并处理”

### 3.5 数据管理页

- 数据管理支持在“主数据库”和“CellData”间切换。
- 左侧标题只显示分类：`主数据库` 或 `CellData`。
- 点击图标按钮弹出数据库列表。
- 弹窗列表显示真实库名，如 `主数据库：CapacityReport`、`CellData：celldata`。
- 表列表、表结构、分页数据、清空、删除、删除全部、导出均跟随当前选择的数据库。

## 5. `cellinfo` 映射规则

目标表：`celldata.cellinfo`

字段：

- `CGI`
- `eNodeBID`
- `CellID`
- `PLMN`
- `基站名称`
- `小区名称`
- `频点`
- `带宽`
- `制式`
- `功率`
- `网络`

`CGI = PLMN-eNodeBID-CellID`

重要修正：

- `LTE_ITBBU_CellInfo` 小区名称字段是 `CellName`
- `LTE_SDR_CellInfo` 小区名称字段是 `cellName`
- 之前 4G 小区名称为空就是因为 SDR 映射错写为 `CellName`。
- 已修正并用 MySQL MCP 验证：4G/5G 的 `小区名称` 空值数均为 0。

默认映射概念：

- `2.6G / LTE_ITBBU_CellInfo`：4G，`制式 <- radioMode`，`网络="4G"`
- `2.6G / LTE_SDR_CellInfo`：4G，`制式 <- radioMode`，`网络="4G"`
- `2.6G / NR_CellInfo`：5G，`制式="2.6G"`，`网络="5G"`
- `700M / LTE_ITBBU_CellInfo`：4G，`制式 <- radioMode`，`网络="4G"`
- `700M / NR_CellInfo`：5G，`制式="700M"`，`网络="5G"`
